MANAGER PORTAL — PAGE / MODAL / POPUP FLOW
=============================================

GLOBAL INFRASTRUCTURE
─────────────────────
  OverlayStackProvider (wraps entire app)
  └── useBackButton hook (global)
      ├── If overlay/step open → close top overlay or step back
      ├── "/" (dashboard) → double-tap to exit
      ├── Sub-pages → navigate to parent
      └── /chats/:id → navigate to /chats

  ConfirmDialog (reusable hook: useConfirmDialog)
  ├── Returns Promise<boolean>
  ├── Renders: backdrop + modal with title, message, confirm/cancel
  └── Used throughout for delete/status-change confirmations

  Toast (Chakra useToast, used on every page)


LAYOUT (wraps all protected routes)
────────────────────────────────────
  ┌─ Sidebar / Top nav (responsive)
  │   ├── Dashboard    → /
  │   ├── Products     → /products
  │   ├── Price Tiers  → /price-tiers
  │   ├── Orders       → /orders  [badge: pending count]
  │   ├── Chats        → /chats   [badge: unread count]
  │   ├── Invites      → /invites
  │   ├── Users        → /users
  │   └── Profile      → /profile
  │
  └─ main: <Outlet /> (scrollable content)


PUBLIC ROUTES
─────────────
  /login → Login page (no modals, toast on error)


PROTECTED ROUTES
────────────────

/ — Dashboard
├── No modals
├── Summary cards: pending orders, active products, unread chats

│   │       │               terpenes, may_relieve, lineage, description
│   │       ├── Image upload (camera with canvas compression, 1200px max)
│   │       ├── Price tier selector
│   │       ├── "Create" button
│   │       └── Back button → Step 2
│   │
│   ├── BATCH CREATE MODE — 4-step wizard
│   │   ├── Step 1: Batch count (5–10)
